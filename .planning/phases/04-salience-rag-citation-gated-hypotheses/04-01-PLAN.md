## Phase Goal

**As an** on-call engineer triaging a production incident, **I want to** run `sift analyze` and get ranked, evidence-cited root-cause hypotheses, **so that** I can trust every claim because it cites a real stored event the model was actually shown — never a fabricated one.

*(Derived faithfully from the ROADMAP Phase 4 goal line; not invented scope.)*

<objective>
Lay the contract + persistence substrate for citation-gated hypotheses: the frozen Pydantic `Hypothesis`/`HypothesisSet` models (SPEC §5.5 verbatim) and store migration 4 (the `hypotheses` table + run-level meta + `replace_hypotheses`/`query_hypotheses`). Purpose: give the pipeline one authoritative schema (whose `model_json_schema()` also feeds constrained decoding) and one durable place to persist hypotheses + degraded-run status.
Output: additive Pydantic models in `models.py`; migration 4 + CRUD + run-meta helpers in `store.py`; round-trip + tamper tests in `test_store.py`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model output → store | Untrusted LLM-generated text (titles, narratives, ids) is persisted to the hypotheses table |
| shared case.db → read path | A case.db can be edited/tampered between runs; query_hypotheses treats every column as untrusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-05 | Tampering (SQLi) | store.replace_hypotheses INSERT | high | mitigate | Parameterised `?` binds only; `_HYP_COLUMNS` is a module constant — no model value ever reaches SQL text (single-SQL-owner invariant) |
| T-04-02 | Spoofing | citations_valid persistence | high | mitigate | Persist the per-hypothesis `citations_valid` flag so an invalid/unverifiable citation is visibly flagged for the report, never silently accepted (backs 04-04 gate) |
| T-04-08 | Tampering | query_hypotheses JSON list columns | medium | mitigate | WR-01 defensive coercion (wrap non-list, str() every element); render-time `_sanitise` in 04-05 strips hostile bytes |
| T-04-SC | Tampering | dependency supply chain | low | accept | No package installs in Phase 4 — all deps already pinned in uv.lock (RESEARCH Package Legitimacy Audit: none added) |
</threat_model>
