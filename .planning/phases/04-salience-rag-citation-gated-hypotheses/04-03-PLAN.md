## Phase Goal

**As an** on-call engineer triaging a production incident, **I want to** run `sift analyze` and get ranked, evidence-cited root-cause hypotheses, **so that** I can trust every claim because it cites a real stored event the model was actually shown — never a fabricated one.

*(Derived faithfully from the ROADMAP Phase 4 goal line; not invented scope.)*

<objective>
Extend the single HTTP boundary so the triage call can request constrained decoding: add an optional `response_format` parameter to `InferenceClient.chat`. Today `chat` only sends `{messages, model}` — Phase 4's enforcement pipeline needs to send the llama.cpp JSON-schema shape so the server constrains output "where available".

Purpose: constrained decoding is the first line of the validate→repair→degrade pipeline (RAG-03); it must live in the one module allowed to open HTTP.
Output: an additive, optional `response_format` param merged into the chat payload; a request-body-shape test asserting the llama.cpp nesting.
</objective>
