<objective>
Build the eustack adapter (INGST-09) for MicroStrategy EU-stack / thread-dump files. Grouping is the genericlog multi-line skeleton with a thread-header record-start trigger instead of a timestamp. The format identity (elfutils `eu-stack` native vs JVM-style dump) was confirmed in 05-02 and decides whether lock/blocked-on attrs exist at all — SPEC says "where present", so extract lock info only if the confirmed fixture carries it. Nothing is fabricated: no per-thread timestamps, no invented severity.

Output: `EustackAdapter`, a sanitised thread-dump fixture (≥2 threads, lock info if the format carries it), and a full test file.

Depends on 05-02: freeze the header/frame/lock regexes against the user-confirmed sample (or the RESEARCH assumed eu-stack shape A3 if the user replied "use assumed shapes"). Keep the grouping rule format-independent so a later fixture correction is a small change.
</objective>

## Phase Goal (MVP user story)

**As an** engineer triaging a MicroStrategy or systemd incident on an air-gapped host, **I want to** run `sift ingest` over journald JSON exports, DSSErrors logs, and EU-stack thread dumps, **so that** real production diagnostics become canonical UTC-ordered events — with nothing dropped silently and multi-node causality never inverted.

**This plan's slice:** a thread dump becomes one citable event per thread, with condensed top frames as the message and lock info surfaced where the format carries it.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-30 | DoS | monster thread block (never-terminated / giant stack) | high | mitigate | 256-line/64 KB caps force-close into a severity="unknown" continuation; streaming parse — bounded memory (Pitfall 5) |
| T-05-31 | Tampering/DoS | malformed / truncated dump (partial thread block) | medium | mitigate | Header-triggered grouping degrades to a severity="unknown" event, byte-accounted; never crash |
| T-05-32 | Tampering/Spoofing | ANSI/terminal escape in frame symbols/message echoed by sift show | medium | mitigate | Existing whole-line _sanitise at render (WR-01) covers the new fields — verified at the Wave-3 e2e slice |
| T-05-33 | Tampering | SQL injection via thread/message content | high | mitigate | Parameterised `?` placeholders only (store.py sole SQL owner); new fields ride the same path |
| T-05-SC | Tampering | package installs | low | accept | Zero external packages — stdlib re/datetime only |
</threat_model>
