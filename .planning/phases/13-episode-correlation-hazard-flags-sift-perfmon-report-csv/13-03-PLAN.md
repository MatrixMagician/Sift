<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| customer CSV header → `attrs` keys | Counter names are attacker-influenceable and become dictionary keys alongside provenance metadata |
| customer CSV rows → `stats.notes` | One malformed header width can drive one note per data row — an unbounded, attacker-chosen multiplier |
| `stats.notes` → `parse_coverage` meta row and stdout | Notes are persisted to the case store and printed to the operator's terminal |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-13-ATTRKEY | Tampering | `_DRIFT_ATTR` / `_RESERVED_ATTRS` in `adapters/dssperfmon.py` | high | mitigate | The new drift marker key is added to `_RESERVED_ATTRS` in the same edit that introduces it, so a counter column crafted with that exact name is routed under `_COUNTER_PREFIX` and cannot overwrite the drift evidence the plan 13-04 hazard reads. Verified by `test_counter_named_like_drift_marker_cannot_shadow_it`. |
| T-13-DOS | Denial of Service | note appends in `adapters/dssperfmon.py` | high | mitigate | `_NOTE_CAP` bounds each note category to a fixed count plus one summary line. Without it, a single header-width mismatch on the reference file yields 13,596 notes — roughly 1 MB in one `parse_coverage` meta row and 13,596 lines to the operator's terminal. Verified by `test_notes_capped`. |
| T-13-DROP | Repudiation | `dict(zip(counter_names, ...))` in `adapters/dssperfmon.py` | medium | mitigate | Collision-qualified keys retain every counter column, so a customer CSV carrying the same counter under two instances no longer silently loses one — an integrity gap that would make the correlator's figures quietly incomplete. Verified by `test_collision_qualified_keys_retain_both_counters`. |
| T-13-EVIDENCE | Repudiation | interaction of WR-02 and WR-05 | medium | mitigate | Capping the notes would destroy the only drift record if the per-event marker did not exist. Tasks 2 and 3 land together and `test_notes_capped` asserts the marker survives the cap, so the disclosure is bounded while the evidence stays complete and citable. |

No package-manager installs occur in this plan, so no `T-13-SC` supply-chain checkpoint applies.
