<objective>
Thread the MCM fact block through `hypothesise` — the chokepoint both `cli.analyze` and
`eval/runner._run_pipeline` reach. Add a sentinel-spliced MCM block to `triage.md` and an
`_apply_mcm_block` mirroring `_apply_kb_block` (residue-free strip → byte-identical no-MCM
prompt); build the facts inside `hypothesise` via `analyse_mcm(store.query_events(), …)` +
`render_mcm_facts`; and — the load-bearing inversion — union the MCM `event_id`s into
`_assemble`'s returned `prompted_ids` so the facts are CITABLE (unlike KB). Prove it with the
determinism / anti-hallucination test and the additivity guard.

Purpose: MCM-06 — structured MCM facts feed `sift analyze` as cited evidence preserving
`cited ⊆ prompted ⊆ store`, with figures computed by the analyser and never authored by the model.
Building inside `hypothesise` (not only `cli.analyze`) is load-bearing: the eval harness calls
`hypothesise` directly, so wiring it here makes the golden case (Plan 03) non-vacuous.
Output: MCM block in triage.md, `_apply_mcm_block`/`_load`-free splice + `_assemble`/`hypothesise`
wiring, the additivity guard, and `tests/test_mcm_analyze.py` (injection + determinism proof).
</objective>
