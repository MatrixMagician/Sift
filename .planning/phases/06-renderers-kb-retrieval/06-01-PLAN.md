<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shared case.db → renderer | Model-generated titles/narratives and adapter-stored raw/message text are attacker-controlled; a shared case.db can be tampered |
| renderer → operator terminal / Markdown file | Rendered bytes reach a terminal, a Markdown viewer, and later WeasyPrint |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Tampering | render_markdown fields (titles, narratives, labels, raw) | high | mitigate | `sanitise` (C0/C1/DEL/Cf strip) on EVERY rendered field/line (WR-01); appendix raw fenced in a code block so log bytes cannot inject Markdown |
| T-06-02 | Spoofing | model narrative `[evt:id]` links | medium | mitigate | Rewrite only ids present in the fetched appendix set; anchors are internal `#evt-…` only; never emit an `http(s)` link from model text (Pitfall 2) |
| T-06-03 | Injection (SQL) | store.get_events_by_ids | high | mitigate | `?`-bound `IN (…)` placeholder list, module-constant column list (S608 convention) — no id ever reaches SQL text |
| T-06-04 | Info disclosure | `sift report` accidentally contacting inference | high | mitigate | report constructs NO InferenceClient and makes no network call — pure function of case.db (verified by absence of client import in render/ and report) |
| T-06-05 | Info disclosure / DoS | corrupt/absent case.db, oversized raw | medium | mitigate | `_case_store` exit-1 sanitised message (no traceback); appendix raw capped + `_decode_raw` zstd-bomb `max_output_size` |
</threat_model>
