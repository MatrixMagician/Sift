<objective>
Deliver the KB retrieval data path (RAG-07): a physically separate KB namespace in `case.db` (migration 5) plus a `pipeline/retrieve.py` that indexes a directory of Markdown runbooks/RCAs and retrieves the nearest chunks by similarity. The separate namespace is the *structural* enforcement of D-01 — KB rows have no `event_id` column, so they can never enter the citable set even if the model tries.

Purpose: Give `sift analyze` a background reference source that demonstrably changes retrieved context, without weakening the load-bearing `cited ⊆ prompted ⊆ store` invariant. This plan builds the store + pipeline seam; 06-04 threads it into the analyze command and prompt.
Output: migration 5 + four confined KB store methods, `pipeline/retrieve.py` (`index_kb`/`retrieve_kb`), ADR 0009 recording the per-case KB location, and a pipeline-level test proving retrieval works and non-citability is structural.
</objective>

## Phase Goal

**As an** on-call engineer, **I want to** point analysis at a directory of our internal runbooks and prior RCAs, **so that** the triage prompt is enriched with relevant background context — while that reference material can never be mistaken for citable case evidence.

<assumption_delta_decision>
The KB index is a SECOND vector namespace alongside the case `vectors` table. This is a deliberate **add-alongside**, NOT a promotion: the primary citable identity stays the case event/vector index; KB is a physically separate, non-promoted, structurally non-citable namespace. Rationale: D-01 requires KB chunks to never enter `prompted_ids`/`supporting_event_ids`; promoting KB into a shared vector model would risk a KB row acquiring an `event_id`. Accepted debt: per-case re-embedding of a shared KB (see ADR 0009 / Assumption A1) — a global index is a clean later switch behind the same store interface. Invariant test that would go red on regression: the D-01 non-citability test in 06-04 (`cited ⊆ prompted ⊆ store` holds even when the model cites a KB chunk).
</assumption_delta_decision>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user KB directory → retrieve.py | `--kb <dir>` walks user-supplied paths; runbook bytes are untrusted text |
| KB vectors → citation gate | KB chunks must be structurally unable to become citable evidence |
| retrieve.py → InferenceClient | KB text is embedded through the sole HTTP boundary (loopback-guarded) |

| T-06-10 | Elevation of privilege | KB chunk masquerading as citable evidence | high | mitigate | `kb_chunks` has NO event_id column and KB ids never enter `prompted_ids` — non-citability is structural (D-01); a shared/flagged table is rejected |
| T-06-11 | Info disclosure | KB directory traversal / reading outside kb_dir | medium | mitigate | `Path.rglob("*.md")` confined to the given dir; UTF-8 decode with error handling; mirror ingest's symlink-skip caution |
