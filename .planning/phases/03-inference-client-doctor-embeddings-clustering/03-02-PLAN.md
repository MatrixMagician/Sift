~200 lines) that talks both inference roles, guards the endpoint against SSRF at construction,
retries with manual backoff, feature-detects llama.cpp extensions without requiring them, and a

guard at the boundary is the load-bearing privacy control.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user/config -> base_url | An attacker-influenced endpoint could point at internal/metadata hosts (SSRF) |
| inference server -> client | Every response body (embeddings, chat text, props) is untrusted input |
| client -> terminal | Server-supplied model IDs / error text may carry hostile control bytes when rendered by callers |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04 | Information Disclosure / Elevation | base_url pointed at public/metadata/internal host (SSRF/exfil) | high | mitigate | Task 1 `_assert_local` refuses non-loopback/non-RFC1918 literals at construction; override is explicit (allow_public); never DNS-resolve; tests assert refusal of 8.8.8.8 |
| T-03-05 | Denial of Service | hostile server streams a huge body / hangs | high | mitigate | explicit httpx timeouts on every request; validate embedding dimension; cap chat/label length; defensive JSON parsing |
| T-03-06 | Tampering | malformed/empty embeddings response (dimension spoof) | medium | mitigate | validate each embedding is a non-empty consistent-length list of floats; raise ValueError on mismatch |
| T-03-07 | Spoofing | server text (model IDs / errors) with control bytes reaches the terminal | medium | mitigate | client returns raw strings; callers (doctor/analyze, Plan 04/06) sanitise at render via cli._sanitise |
</threat_model>
