<objective>
Extend the case store with the persistence surface for embeddings and clusters: migration 3
(chunks + clusters tables), a lazily-created sqlite-vec `vec0` vectors table whose dimension is
discovered from the server's first embedding, the STORE-03 dimension guard, the confined float32
(de)serialisation, and caller-owns-transaction `replace_clusters` / `upsert_vectors` methods.

Purpose: STORE-03 (identity + dimension in meta, mismatch = hard error) is a load-bearing
correctness item — a wrong dimension silently re-indexing would corrupt every downstream cluster.
Keeping all vector bytes inside store.py preserves the documented BLOB+numpy escape hatch.

Output: `src/sift/store.py` with migration 3, `ensure_vectors_table`, `_vec_to_blob`/`_blob_to_vec`,
`upsert_vectors`, `replace_clusters`, and cluster/chunk query methods; `tests/test_store_vectors.py`.
</objective>

| sqlite-vec extension -> process | Native extension loading is a code-execution surface |
| server dim -> vectors table | A wrong/spoofed dimension would corrupt the index |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
