## Phase Goal

**As a** Sift maintainer, **I want to** run `sift eval` over a committed golden-case suite that scores hypothesis quality and fails when scores regress below `eval/thresholds.toml`, **so that** hypothesis quality is measurable and regression-gated rather than vibes-based.

This plan makes `sift eval` **CI-usable**: it turns the printed metrics (Plan 02) into a pass/fail gate with a non-zero exit — the primary SPEC §8 / ROADMAP SC3 acceptance.

<objective>
Add per-metric floors (`eval/thresholds.toml`), a threshold gate, and the CI-friendly non-zero exit (D-07, EVAL-03). Resolve the determinism direction gotcha by gating on `determinism_stability` (higher-is-better, floor 1.0) while the report DISPLAYS drift = 1 − stability, and record the exit-code contract in ADR 0010.

Purpose: a planted regression must fail the suite with exit 1 (SPEC §8 acceptance); a clean suite exits 0.
Output: thresholds file + loader + gate + CLI exit wiring + ADR 0010 + the planted-regression test.
</objective>
