<objective>
Complete STORE-04: the user can inspect stored data via `sift show <case> events|clusters
[--filter key=value]...` before trusting any AI output. Filter keys are allowlisted per target
and mapped to fixed parameterised WHERE snippets inside store.py (the single SQL owner,
Phase 1 T-02-02 invariant); `show events` switches to a column-scoped streaming query so a
1M-event case renders without hydrating full Events or decompressing raw.

Purpose: inspectability is the trust foundation for every later AI phase — and the filter
surface is the phase's main injection boundary, so it lands as its own reviewed slice.
Output: filtered queries in store.py, --filter on the show command, exhaustive filter tests.
</objective>

## Phase Goal

**As an** engineer triaging a production incident offline, **I want to** ingest a large log
collection into one portable case file and see it collapse into inspectable template groups,
**so that** I can understand what the log contains at a glance — with zero LLM dependency and
nothing dropped silently.
