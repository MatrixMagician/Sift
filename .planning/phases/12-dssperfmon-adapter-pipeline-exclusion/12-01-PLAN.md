  <action>
    Create `src/sift/adapters/dssperfmon.py` mirroring `dsserrors.py`'s module layout. Import
    `csv` from the stdlib in place of `re` (D-20) — no regular expression is used anywhere in this
    module, which discharges the ReDoS threat by construction.

    Module constants: `PDH_SNIFF_PREFIX: bytes` holding the anchored literal PDH marker used by the
    sniff, and `TS_FORMAT: str = "%m/%d/%Y %H:%M:%S.%f"` for `datetime.strptime` (D-12).

    `class DssperfmonAdapter(ConfigurableAdapter)` with `name = "dssperfmon"` (D-04). Its docstring
    states the PERF-01/PERF-02 requirement ids and, following the `DsserrorsAdapter` docstring
    convention, why per-run config travels on the instance.

    `sniff(self, path: Path) -> float` returns `0.95` when `read_head(path)` begins with
    `PDH_SNIFF_PREFIX`, else `0.0`. Compare bytes directly — no decode, no scan, no regex. The
    high-but-unique score matters because `detect()` scores every registered adapter against every
    file and the existing maximum is `dsserrors`'s `0.8` (Pitfall 3).

    `parse(self, path: Path, case_id: str) -> Iterator[Event]` opens the parse preamble exactly as
    `dsserrors.parse` does: derive `relpath` from `self.input_root`, obtain `override_tz` via
    `tz_override_for(relpath, self.tz_overrides)`, create `stats = ParseStats(path=relpath)`, and
    initialise `offset = 0`, `line_no = 0`, `header_columns: list[str] | None = None`.

    The read loop iterates `byte_lines(stream, b"\n", b"", unit=1)` inside
    `with open_bytes(path) as stream`. Per iteration, capture `line_offset = offset`, then advance
    `offset += len(bline)` BEFORE any decoding — no parse outcome may perturb the offset, and this is
    what keeps `event_id` deterministic and re-ingest idempotent. Decode with
    `errors="replace"`, strip trailing CR/LF for parsing while keeping the undecoded-length
    accounting on `bline`. A line that is empty after stripping advances the offset and continues.
    Parse each row with a single-row call to `csv.reader` over a one-element list of the decoded
    text — stdlib csv must never own file iteration (Pitfall 4).

    The first parsed row is the header: store it in `header_columns` and continue without emitting an
    Event (D-01). Pass it to `_parse_header(header_columns)`, a module function returning the host,
    the declared zone name, the declared numeric bias as a string, and the list of short counter
    names. It derives the host and each short name from column paths of the form
    `\\host\Object(Instance)\Counter(unit)` by splitting on the backslash separator: the host is the
    first non-empty segment, the short name is the final segment including its parenthesised unit
    (D-02). Extract the zone name and bias from the parenthesised groups of `header_columns[0]`
    using string partitioning on the parentheses, not a regex. Record the "declared bias recorded,
    not applied" disclosure once in `stats.notes`, citing ADR 0012.

    `_short_counter_name(path: str) -> str` is the helper doing the final-segment extraction, so
    `_parse_header` stays readable and the behaviour is unit-testable.

    For each subsequent row, build the Event: `datetime.strptime(row[0], TS_FORMAT)` produces a naive
    datetime which is handed to `to_utc(dt, override_tz)` — the identical call shape `dsserrors`
    uses — and its returned pair supplies `ts` and `ts_confidence`. ADR 0012 is binding here: the
    declared bias from the header is written into `attrs` as `tz_name` and `tz_offset_min` and is
    never used in any arithmetic on the datetime. `severity` is unconditionally `"info"` for a
    well-formed row (D-05); the adapter forms no judgement about counter magnitude. `component` is
    the header-derived host. `thread` and `session` are `None`. `line_start` and `line_end` are both
    the current `line_no` (D-04). `event_id` comes from `models.event_id(relpath, line_offset)`.
    `raw` is the verbatim decoded line. `message` is the deterministic short-name `name=value`
    rendering joined by a stable separator. `attrs` is a `dict[str, str]` holding `byte_offset` and
    `byte_len` — those exact key spellings are load-bearing, as the reused `assert_span_partition`
    helper reads them — plus `host`, `pdh_version` set to `"4.0"`, `tz_name`, `tz_offset_min`, and
    one entry per counter keyed by short name with the cell value as an unconverted string (D-03;
    numeric interpretation is Phase 13's job). Increment `stats.event_count` per emitted Event.

    After the loop, set `stats.total_bytes = offset` and `self.last_stats = stats`, so
    `ParseStats.coverage` is correct without new machinery.

    Malformed-row handling (blank cells, unparseable timestamps, column-count drift) is deliberately
    NOT in this task — plan 12-02 adds those paths. Write this task's code so a `strptime` failure or
    a short row raises nothing catastrophic in the fixture path, and leave the fallback branches to
    12-02 rather than stubbing them.

    Create `tests/test_dssperfmon.py` mirroring `tests/test_dsserrors.py`: module docstring naming the

    "Artifacts this phase produces" above, covering every bullet in the behaviour block. Type-annotate
    everything; the project runs pyright strict.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| customer artefact → adapter | The PDH-CSV is untrusted third-party input read from disk; contents, sizes, encodings, and counter names are all attacker-influenceable in principle |
| compressed file → decompressed stream | `base.open_bytes` handles gzip/zstd; a hostile archive crosses here |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-01 | Denial of Service | `DssperfmonAdapter.sniff` | medium | mitigate | Anchored literal byte-prefix comparison against `PDH_SNIFF_PREFIX`; no regex is imported by the module, so crafted counter names cannot trigger catastrophic backtracking |
| T-12-02 | Denial of Service | `DssperfmonAdapter.parse` read loop | medium | mitigate | `genericlog.byte_lines` force-splits at `MAX_EVENT_BYTES` (already shipped), bounding per-line memory for a single-monster-line file |
| T-12-03 | Tampering | timestamp normalisation | medium | mitigate | Timestamps flow only through the shared `base.to_utc` seam; the file-declared bias is stored as inert `attrs` metadata and never enters arithmetic (ADR 0012), so a crafted header cannot shift a case's timeline |
| T-12-04 | Information disclosure | adapter module | low | accept | The adapter performs pure local parsing and opens no socket; the autouse `_no_network` conftest guard enforces this in tests |
| T-12-05 | Denial of Service | `base.open_bytes` decompression | low | accept | Existing shipped behaviour, unchanged by this phase; no new exposure is introduced |
| T-12-SC | Tampering | dependency supply chain | high | mitigate | No package-manager install occurs: the parser uses stdlib `csv` only (D-20). Adding a dependency would require a fresh Package Legitimacy Gate and a blocking human checkpoint |
</threat_model>
