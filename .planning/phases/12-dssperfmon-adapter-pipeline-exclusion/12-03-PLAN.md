<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arbitrary case file → detect() | Every file in an ingested case directory is now scored by a fifth sniff; untrusted content crosses into `DssperfmonAdapter.sniff` |
| CLI argument → adapter selection | `--adapter` and config globs can name an adapter, but the default path is content-based routing |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-10 | Denial of Service | `detect()` sniff loop | medium | mitigate | The added sniff is a fixed-length anchored byte-prefix comparison over `read_head` output — constant work per file, no scanning and no regex, so the fifth adapter adds bounded cost |
| T-12-11 | Tampering | adapter routing | high | mitigate | A crafted file that hijacked routing would silently change how customer diagnostics are parsed; the unique-maximum assertion in `test_phase5_no_cross_collision` is the gate, extended here to cover all five adapters |
| T-12-12 | Spoofing | `REGISTRY` contents | low | accept | Registration is a source-level dict with no dynamic or plugin loading; only a code change can add an adapter |
| T-12-13 | Information disclosure | CLI tests | low | mitigate | The autouse `_no_network` conftest guard raises on any socket connect, so CLI-level tests cannot reach a server |
| T-12-SC | Tampering | dependency supply chain | high | mitigate | No package-manager install occurs in this plan |
</threat_model>
