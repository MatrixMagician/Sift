<objective>
Complete the `dssperfmon` adapter's nothing-disappears-silently contract: every malformed row —
bad cell, bad timestamp, wrong column count, embedded newline — becomes a `severity="unknown"`
Event whose bytes are reflected in per-file parse coverage. Then pin ADR 0012 with the
cross-artefact alignment test that is the only test able to catch the 5-hour skew.

Purpose: PERF-02 and roadmap criterion 3. The reference CSV contains zero malformed cells (D-17),
so these paths exist only under synthetic fixtures — which is precisely why they must be authored
deliberately rather than assumed.
Output: the fallback branches in `src/sift/adapters/dssperfmon.py` plus their tests.
</objective>

  <behavior>
    - Blank counter cell: the row still yields exactly one Event; `severity == "unknown"`; `attrs["unparsed_columns"]` names the offending short counter name; `raw` equals the input line verbatim; `ts` is still parsed normally.
    - Non-numeric counter cell (e.g. `N/A`): same outcome as the blank case, same `unparsed_columns` mechanism.
    - Unparseable timestamp: one Event with `severity == "unknown"`, `ts is None`, `ts_confidence == "missing"`, `raw` verbatim.
    - Column count below or above the header's: one Event with `severity == "unknown"`, `raw` verbatim, and a `ParseStats.notes` entry naming the line number and the observed versus expected count.
    - Embedded newline inside a quoted field: two Events, both `severity == "unknown"`, and `assert_span_partition` still passes.
    - Header lacking a parseable zone or bias: rows parse normally with `ts_confidence == "inferred"`, and a `ParseStats.notes` entry records the absent declaration.
    - Coverage: for a fixture of N good rows and M unknown rows, `stats.unknown_fallback_bytes` equals the summed byte length of the M rows and `stats.coverage` is below 1.0.
  </behavior>
  <action>
    Extend the per-row handling in `DssperfmonAdapter.parse` with the three malformed branches, keeping
    the read loop and the offset accounting from 12-01 untouched — no branch may alter, skip, or
    re-order the `offset` advance.

    Introduce a module-level `_fallback_event(...)` helper that builds an Event with
    `severity="unknown"`, `raw` set to the verbatim decoded line, the same `byte_offset` / `byte_len`
    attrs spellings, and the caller-supplied `ts` / `ts_confidence` / extra attrs. Every fallback
    branch routes through it, so there is one place where the never-drop guarantee is enforced.

    Branch order inside the row handler:

    0. **Attempt the timestamp parse on `row[0]` BEFORE any fallback branch, and carry the result
       into whichever branch fires.** A malformed row can still carry a perfectly good timestamp,
       and the timestamp is what lets Phase 13 place the surviving evidence on a timeline. D-16
       requires only `severity="unknown"` plus a notes entry — it does not require discarding a `ts`
       that parsed cleanly. Emitting `ts=None` unconditionally from the column-count branch would
       throw away recoverable evidence, and an embedded-newline record would lose its timestamp on
       *both* fragments.

    1. Column-count check first (D-16). Compare the parsed row length against the stored header
       length. On mismatch, emit via `_fallback_event` passing through the `ts` / `ts_confidence`
       from step 0 when the parse succeeded (falling back to `ts=None` / `ts_confidence="missing"`
       only when it genuinely did not), and append a `stats.notes` entry naming the line number, the
       observed count, and the expected count. Severity is `"unknown"` either way. Do not attempt to
       realign, pad, or truncate the row — surviving drift is Phase 12's job; diagnosing it is
       PERF-05's, in Phase 13.
    2. Timestamp parse (D-15). Wrap the `strptime` call in a `try` and treat `ValueError` as the
       fallback trigger, mirroring the `except ValueError: return None` guard in `dsserrors._match_ts`.
       Emit via `_fallback_event` with `ts=None` and `ts_confidence="missing"`, still populating the
       counter attrs from whatever columns did parse.
    3. Cell validation (D-14). Scan the counter cells for values that are empty, whitespace-only, or
       fail a `float()` conversion. Collect the short counter names of the failures. If the collection
       is non-empty, emit via `_fallback_event` retaining the successfully parsed `ts` and
       `ts_confidence`, and set `attrs["unparsed_columns"]` to the failing short names joined by a
       stable separator. The `float()` attempt is a validity probe only — the cell values themselves
       stay unconverted strings in `attrs`, per D-03.

    In all three branches set the same per-row fallback flag that causes the row's byte length to be
    added to `stats.unknown_fallback_bytes`, matching how `dsserrors.finish()` accumulates. This is
    what makes `ParseStats.coverage` reflect the loss without any new counter (PERF-02).

    In `_parse_header`, make the zone-name and bias extraction tolerant: when `header_columns[0]`
    carries no parenthesised zone or no numeric bias, omit the corresponding `attrs` key rather than
    inventing a value, and disclose that the declaration was absent. Nothing about the timestamp path
    changes — ADR 0012 means the bias is inert metadata, so its absence cannot affect any `ts`.

    **Preserve 12-01's `_parse_header(header_columns)` signature — do NOT add a `stats` parameter.**
    12-01 established that signature and the caller owns `stats`. Have `_parse_header` return the
    disclosure strings as an additional element of its return tuple, and let the existing caller
    append them to `stats.notes`. Threading `stats` into the function instead would be a silent
    cross-plan signature change to a wave-1 artifact.

    Take no action on embedded newlines. `byte_lines` splits on the newline byte unconditionally, so
    such a record arrives as two fragments, each hitting the column-count branch and becoming its own
    unknown Event with bytes preserved and offsets intact. RESEARCH Q4 establishes that reassembly
    would compromise the offset contract for a case the PDH format cannot produce; the test in Task 2
    pins the split behaviour so it is deliberate rather than accidental.

    Add the synthetic tests named in "Artifacts this phase produces", authoring each malformed input
    inline through the `write` helper so the defect under test is visible in the test body rather than
    hidden in a checked-in file. Call `assert_span_partition` in every one of them.
  </action>
