<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `case` CLI argument → filesystem path | A user-supplied case name must never escape the configured data directory |
| `--format` CLI argument → control flow | An unknown value must be rejected by the framework, not by hand-rolled parsing |
| exception text → operator stdout | `OSError` messages embed filesystem paths and may embed customer-derived names |
| hazard messages → operator stdout | Hazard text embeds counter names originating in the customer CSV header |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-13-PATH | Tampering | `perfmon` command bundle directory | high | mitigate | The bundle directory is `case_db_path(config.data_dir, case).parent / "perfmon"`, derived from the path `_case_store` already validated via `validate_case_name` and `case_db_path`'s containment assertion. No user-supplied path is ever joined, so a traversal payload in `case` cannot direct the write. Verified by `test_missing_case_exit_one` and by reusing the proven `sift mcm` line. |
| T-13-ERRLEAK | Information Disclosure | `except OSError` branch | high | mitigate | The failure path prints `_sanitise(str(exc))` and raises `typer.Exit(1) from None`. `from None` suppresses the traceback chain so no stack frame, local variable or internal path reaches the operator, and `_sanitise` strips control and Cf characters from the message. Verified by `test_write_failure_exit_one` asserting no `Traceback` in the output. |
| T-13-STDOUTESC | Tampering | stdout summary block | medium | mitigate | Every hazard message is passed through `_sanitise` before echo, mirroring the `sift mcm` precedent. A counter name carrying a C1 CSI byte or bidi override therefore cannot drive the operator's terminal from the summary line. |
| T-13-FMTPARSE | Tampering | `--format` handling | low | mitigate | `PerfmonFormat` is a `StrEnum` parameter, so an unknown value is rejected by Typer as a usage error (exit 2) before any code in the command body runs and before any filesystem access. No hand-rolled format parsing exists. Verified by `test_exit_codes`. |
| T-13-WAL | Denial of Service | store lifetime | low | mitigate | `store.close()` runs in a `finally` covering every path including each `typer.Exit`, so the WAL checkpoints and no case is left with an unbounded write-ahead log after a failed run. |
| T-13-DOUBLEREAD | Denial of Service | `store.query_events()` | low | mitigate | The method hydrates and zstd-decompresses every row in the case. It is called exactly once and its result feeds both `analyse_mcm` and `analyse_perfmon`, so the accepted single-pass cost is not doubled. Asserted by the occurrence-count criterion in task 2. |

No package-manager installs occur in this plan, so no `T-13-SC` supply-chain checkpoint applies.
</threat_model>
