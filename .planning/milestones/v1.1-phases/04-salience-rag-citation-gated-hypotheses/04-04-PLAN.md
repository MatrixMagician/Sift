## Phase Goal

**As an** on-call engineer triaging a production incident, **I want to** run `sift analyze` and get ranked, evidence-cited root-cause hypotheses, **so that** I can trust every claim because it cites a real stored event the model was actually shown — never a fabricated one.

*(Derived faithfully from the ROADMAP Phase 4 goal line; not invented scope.)*

<objective>
Build the load-bearing core: `pipeline/hypothesise.py` — assemble the budgeted triage prompt (tracking the prompted event-id universe), run the enforcement state machine (constrained decode → Pydantic validate → one repair → degrade), then the citation gate (cited ⊆ prompted, regenerate once, flag), and persist everything atomically. Plus the versioned `prompts/triage.md` and the fake OpenAI-compatible server fixture returning the canned scenarios.

Purpose: this is the anti-hallucination mechanism — a hypothesis cannot cite what the model was never shown, and malformed output degrades instead of crashing.
Output: `hypothesise(...)` returning a structured `Outcome`; `triage.md`; `test_hypothesise.py` covering repair, degrade, regenerate, flag, citation-valid, and determinism with a fake server.
</objective>
