<objective>
Build the dsserrors adapter (INGST-08) for MicroStrategy `DSSErrors.log` and its rotated `.bak00`/`.bak01` siblings. This is the closest structural clone of genericlog: timestamp-parse + shared UTC/tz-override path AND multi-line grouping-with-caps. Token extraction anchors on the version-stable signatures (`[*.cpp:NNNN]`, `0x` codes, GUID OIDs, MCM `***** Start/End of Info Dump *****` sentinels) rather than on column order. Node attribution and mixed-timezone timeline ordering (criterion 4) both land here.

Purpose: deliver INGST-08 and the cross-cutting criterion-4 UTC-timeline guarantee at adapter level.
Output: `DsserrorsAdapter`, sanitised multi-node fixtures with reversed-chronology `.bak` siblings, and a full test file.

Depends on 05-02: freeze the record-line layout and SID token shape against the user-confirmed sample (or the RESEARCH assumed shapes A1/A2 if the user replied "use assumed shapes"). Keep extraction token-anchored so a later fixture correction stays a small change.
</objective>

## Phase Goal (MVP user story)

**As an** engineer triaging a MicroStrategy or systemd incident on an air-gapped host, **I want to** run `sift ingest` over journald JSON exports, DSSErrors logs (and their rotated `.bak` siblings), and EU-stack thread dumps, **so that** real production diagnostics become canonical UTC-ordered events — with nothing dropped silently and multi-node causality never inverted.

**This plan's slice:** a multi-node DSSErrors bundle — SIDs, OIDs, error codes, MCM dumps, rotated siblings, mixed timezones — becomes a correctly ordered UTC timeline where causality is never silently inverted.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted DSSErrors bytes → parse() → Event → store/show | Proprietary log content is untrusted input treated strictly as data |
| crafted node directory name → attrs["node"] | A hostile bundle path must never select or open a file outside the input root |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-20 | DoS | never-terminated MCM block (SIGKILL-truncated dump) | high | mitigate | 256-line/64 KB caps force-close into a severity="unknown" continuation; streaming parse — a truncated block cannot slurp unbounded memory (Pitfall 5) |
| T-05-21 | Tampering | path traversal via crafted node directory name | medium | mitigate | node derived from the already-validated case-relative path (symlinks skipped by cli.py); it is metadata only, never used to open a file (Pitfall 3 boundary) |
| T-05-22 | Tampering | SQL injection via SID/OID/message content | high | mitigate | Parameterised `?` placeholders only (store.py is the sole SQL owner); new fields ride the same path unchanged |
| T-05-23 | Tampering | severity token mapped outside the 6-value CHECK rolls the file back | high | mitigate | Exhaustive severity map defaults to "unknown"; test each tag (Pitfall 4) |
| T-05-24 | Tampering/Spoofing | ANSI/terminal escape in message/session/thread echoed by sift show | medium | mitigate | Existing whole-line _sanitise at render (WR-01) covers the new fields — verified at the Wave-3 e2e slice |
| T-05-SC | Tampering | package installs | low | accept | Zero external packages — stdlib re/datetime only |
