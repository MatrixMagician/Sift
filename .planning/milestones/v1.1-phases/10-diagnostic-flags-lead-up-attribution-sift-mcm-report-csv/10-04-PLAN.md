<objective>
Ship MCM-05: the `sift mcm <case>` command that turns the Plan-03 `McmAnalysis`
into a durable forensics bundle — a deterministic human-readable report
(Markdown default → `mcm_report.md`, `--format json` → `mcm_report.json`) and a
single dimension-tagged CSV (`mcm_attribution.csv`) — always written into
`<case>/mcm/`, plus a short stdout summary (D-10). The report is timeline-first
per episode (D-11); the CSV carries every attribution row's owning `event_id`s
(D-15/D-16). The renderer is a pure `McmAnalysis→str` mirroring `render/markdown.py`,
routing every attacker-controlled log field through the shared `sanitise`/`_field`
escaping; only the CLI tier touches the filesystem.

Purpose: this is the one-shot bundle the engineer runs; it is also the security
boundary of the phase (hostile DSSErrors text flowing into Markdown and CSV).
Output: `render_mcm_markdown`/`render_mcm_json`/`write_attribution_csv` +
the `sift mcm` command.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dsserrors log text → Markdown/JSON report | Untrusted `Source=`/SID/OID/lifecycle/raw text rendered into a shareable report (and the PDF-capable HTML path via markdown) |
| dsserrors log text → CSV export | Untrusted key/Source values written into a spreadsheet-openable CSV |
| `case` argument + write target | User-supplied case name resolved to a filesystem path for the `<case>/mcm/` bundle |
| stdout summary | Log-derived text echoed to the operator's terminal |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-12 | Tampering | Markdown/HTML injection via hostile log text into `mcm_report.md` | high | mitigate | Route every log-sourced field through the shared `sanitise` + `_field`/`_escape` (markdown.py precedent); table cells escape `|`; raw/breakdown text fenced/escaped — hostile bytes cannot inject headings, links, fake markers or raw `<img>` into the report or PDF path. `test_markdown_sanitises_hostile_fields` is the guard |
| T-10-13 | Tampering | CSV injection / formula injection / row-break via `key`/`Source` fields | high | mitigate | stdlib `csv.writer(newline="")` quotes embedded delimiters/quotes/newlines; CSV keys are structurally hex (SID/OID) or `Source=` `[\w:]+` values which cannot begin with a `= + - @` formula trigger — documented; never manual-join |
| T-10-14 | Tampering | path traversal on the `<case>/mcm/` write | high | mitigate | Derive the bundle dir from `case_db_path(config.data_dir, case).parent` (the same resolution `_case_store` validates); create only `<case>/mcm/` beneath it; never join a user-supplied path outside the resolved case dir (RESEARCH Security V12) |
| T-10-15 | Tampering | terminal-injection via the stdout summary | medium | mitigate | Pass any log-derived summary text through `_sanitise` (strips C0/C1/DEL/Cf), the CLI precedent for echoing store-sourced text |
| T-10-16 | Information Disclosure | absolute-GB figure leaking into a report headline | high | mitigate | Flags/window framed as % of HWM/total in the renderer; only the attribution table/CSV carries absolute `granted_bytes`/`granted_mb` (legitimately, per RESEARCH Deliverable 3); `test_markdown_no_absolute_gb_headline` guards the headline framing |
| T-10-17 | Denial of Service | render of a very large/open-truncated episode | low | mitigate | Renderer is a pure pass over the bounded `McmAnalysis`; raw/breakdown text follows the markdown byte-cap/fence precedent; no unbounded recursion |
| T-10-SC | Tampering | dependency installs | low | accept | No package installs — stdlib `csv`/`json` + already-installed Pydantic/Typer only; supply-chain surface unchanged |
</threat_model>
