<objective>
Deliver PKG-02 (D-05, D-06 corrected, D-07, D-08): ship the optional Podman Quadlet deploy files with a llama-server example, and lock the single load-bearing invariant — the in-container Sift reaches a host-side llama-server through the DNS-free SSRF guard (LLM-02) without `--i-know-what-im-doing`. Record the corrected mechanism in ADR 0011.

Purpose: the guard (`src/sift/llm/client.py:54-84`) inspects the literal hostname and never resolves DNS. A bare host magic-DNS name is therefore rejected; a literal loopback IP is accepted. The deploy default must ship a guard-clean address (`Network=host` + `http://127.0.0.1:<port>/v1`) and a test must lock it against regression.
Output: `deploy/sift.container`, `deploy/llama-server.container.example`, `docs/decisions/0011-quadlet-loopback-guard.md`, and PKG-02 tests appended to `tests/test_packaging.py`.
</objective>
