<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shared case.db → JSON renderer | Model/adapter text is attacker-controlled; a tampered case.db can hold arbitrary JSON in list columns |
| JSON document → downstream tooling | Emitted JSON is consumed by other tools; must not leak host paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-06 | Info disclosure | render_json run-metadata block | medium | mitigate | No absolute path or cwd embedded; Event.source_file is case-relative; `normalise_for_determinism` also drops any stray absolute path defensively |
| T-06-07 | Tampering | list columns (supporting_event_ids, unexplained_signals) from a tampered case.db | low | mitigate | Store coerces list columns to `str` defensively (query_hypotheses/query_clusters precedent); JSON encodes strings, injects no executable content |
| T-06-08 | Info disclosure | `sift report --format json` contacting inference | high | mitigate | render_json constructs no client and makes no network call — pure store function |
| T-06-09 | Repudiation | determinism claim overstated | low | accept | ADR 0008 scopes the claim to the renderer; live-backend bit-exactness is explicitly NOT guaranteed (D-07) |
</threat_model>
