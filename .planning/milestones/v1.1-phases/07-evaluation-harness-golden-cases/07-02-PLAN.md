## Phase Goal

**As a** Sift maintainer, **I want to** run `sift eval` over a committed golden-case suite that scores hypothesis quality and fails when scores regress below `eval/thresholds.toml`, **so that** hypothesis quality is measurable and regression-gated rather than vibes-based.

> Story derived from the ROADMAP Phase 7 `**Goal:**` line ("Hypothesis quality is measurable and regression-gated, not vibes-based") + 07-CONTEXT domain statement. The ROADMAP goal predates mvp-phase user-story formatting; no scope is invented — every clause maps to ROADMAP SC1–SC4.

This plan is the **first vertical slice / walking skeleton of the harness**: one golden case flows input → ingest → cluster → hypothesise → metrics → printed row, entirely offline. Breadth (all 6 cases), the threshold gate, and the judge are added by Plans 03/04/05.

<objective>
Stand up `src/sift/eval/` and fill the `sift eval` stub so ONE golden case runs end-to-end offline and prints its metric row. Every metric is a pure read of already-persisted `case.db` rows + the frozen `truth.yaml` (RESEARCH: this phase adds measurement, not pipeline behaviour).

Purpose: deliver the thinnest working `sift eval` a user can run and see quality numbers for (EVAL-02) plus the truth.yaml schema (EVAL-01, partial).
Output: the eval package, a filled CLI command (exit 0, no gate yet), one committed golden case, and offline tests.
</objective>
